# 0.0.3 — Config/Profiles hardening + effective-config artifacts + release gate stabilization

Release target: `0.0.3`.
Previous released version: `0.0.2`.

This release also aligns public and native-facing version metadata on the `0.0.3` line so the repository no longer presents `0.1.0` as the current public release.

Native CLI work may continue in parallel, but `0.0.3` itself remains the OSS-readiness release. The current public release still includes the Python implementation and Python packaging path.

This release focuses on making the skill easier to operate deterministically:

- Profiles are treated as first-class overlays for pipeline experiments.
- Effective config artifacts are written to stable paths for audit/debug.
- Release check Phase2 (doctor + pytest + topic smoke) is stabilized and produces pinned reports.

No server runtime is included (by design).

## Highlights

- Profiles
  - `--profile` shorthand prefers `.kano/backlog_config/<ref>.toml` first.
  - Fallback for shorthand: `<repo_root>/<ref>.toml`.
  - Explicit paths still work (absolute, `./...`, or values ending in `.toml`).
- Effective config artifacts
  - Stable: `.kano/cache/effective_backlog_config.toml`
  - Runtime overrides: `.kano/cache/effective_runtime_backlog_config.toml`
- Env auto-load for local dev
  - Default: load `env/local.secrets.env` when present.
  - Override: `--env-file` or `KANO_ENV_FILE`.
  - Never overwrite an already-set environment variable.
- Embedding providers
  - Gemini embeddings via google-genai are supported (see profile `profiles/embedding/gemini-embedding-001.toml`).
